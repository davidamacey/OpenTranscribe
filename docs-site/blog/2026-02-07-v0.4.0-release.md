OpenTranscribe v0.4.0 is a major enterprise-focused release bringing production-grade security, cutting-edge AI improvements, and intelligent search capabilities to the platform. This release transforms OpenTranscribe into a comprehensive solution for organizations of all sizes, with enterprise authentication, advanced speaker diarization, and neural search powered by OpenSearch.

## What's New in v0.4.0

### Enterprise Authentication System

OpenTranscribe now supports **four authentication methods** that can run simultaneously, enabling seamless integration into any enterprise environment:

**Authentication Methods:**

1. **Local (Direct)** - Username/password with bcrypt hashing and optional MFA
2. **LDAP/Active Directory** - Enterprise directory integration for seamless user management
3. **OIDC/Keycloak** - OpenID Connect support for external identity providers
4. **PKI/X.509** - Certificate-based authentication for high-security environments

**Key Features:**

- **Zero-Restart Configuration** - Change authentication settings via the Admin UI without restarting services
- **Hybrid Mode** - Enable multiple auth methods simultaneously; users can choose their login method
- **Multi-Factor Authentication (MFA)** - TOTP-based MFA compatible with Google Authenticator, Authy, Microsoft Authenticator, and all standard authenticator apps
- **Super Admin Configuration UI** - Intuitive settings page for configuring each auth method
- **Password Policies** - Enforce minimum length, complexity, character requirements, and expiration
- **Password History** - Prevent reuse of recent passwords
- **Account Lockout** - Automatic lockout after configurable failed login attempts
- **Rate Limiting** - Per-IP and per-user rate limiting to prevent brute force attacks
- **Audit Logging** - Complete authentication event logging for compliance

**Enterprise Benefits:**

- **Seamless Integration** - Works with existing corporate LDAP/Active Directory without additional infrastructure
- **No Credential Management** - OIDC and PKI users delegate authentication to trusted identity providers
- **Regulatory Compliance** - Comprehensive audit trails and security controls meet enterprise requirements
- **User Self-Service** - MFA setup, password management, and session control available to users

**Getting Started:**

```bash
# Use the development setup with LDAP test container
./opentr.sh start dev --with-ldap-test

# Or with Keycloak OIDC
./opentr.sh start dev --with-keycloak-test

# Or production PKI for certificate authentication
./opentr.sh start prod --build --with-pki
```

Access the authentication settings in the Admin UI: **Settings → Authentication**

See full documentation in:
- `docs/LDAP_AUTH.md` - LDAP/Active Directory setup
- `docs/KEYCLOAK_SETUP.md` - OIDC/Keycloak integration
- `docs/PKI_SETUP.md` - Certificate authentication

### PyAnnote v4 Migration: Speaker Diarization Revolution

OpenTranscribe now uses **PyAnnote v4**, delivering massive performance improvements and new capabilities for speaker identification:

**Performance Gains:**

- **273x faster speaker assignment** - Drastically reduced diarization processing time
- **40-60 second startup savings** - Warm model caching eliminates cold-start delays
- **Better speaker overlap detection** - Improved accuracy for multi-speaker conversations
- **Reduced memory footprint** - More efficient GPU utilization

**New Features:**

- **Speaker Overlap Detection** - Automatically identifies and marks moments where multiple speakers talk simultaneously
- **Confidence Scoring** - Speaker identification includes reliability metrics
- **Cross-Video Speaker Matching** - Intelligently matches speakers across multiple recordings using voice embeddings
- **Speaker Merge Interface** - Easily combine duplicate speaker instances detected across videos

**Migration Path:**

The migration is automatic and transparent:

```bash
# Simply restart the backend with the new version
docker compose pull
docker compose up -d backend

# Or use the development script
./opentr.sh restart-backend
```

**Backward Compatibility:**

- All existing diarization data remains accessible
- Videos process with improved accuracy using v4
- No manual intervention required
- Speaker assignments from previous versions are preserved

**What This Means for Users:**

- **Faster Processing** - Videos finish transcription much quicker
- **Better Speaker Detection** - More accurate identification of who's speaking
- **Better Overlap Handling** - When multiple people speak simultaneously, it's now detected
- **Efficient Resource Usage** - Better performance on limited hardware

### OpenSearch Neural Search: Intelligent Hybrid Search

OpenTranscribe v0.4.0 introduces **OpenSearch neural search**, combining traditional full-text search with AI-powered semantic search for dramatically better search results:

**What is Neural Search?**

Neural search uses machine learning models to understand the *meaning* of your search query and documents, not just keyword matching. Search for "meeting decisions" and find segments discussing conclusions, even if those exact words aren't present.

**Hybrid Search Approach:**

OpenTranscribe combines two search methods for optimal results:

- **BM25 Full-Text Search** - Fast, traditional keyword matching; excellent for exact phrase searches
- **Neural/Semantic Search** - Semantic understanding; great for intent-based queries and concept matching

Results combine both approaches, ensuring you find exactly what you're looking for whether you use specific keywords or natural language queries.

**Search Improvements:**

- **Concept Matching** - Find related content even with different wording
- **Multi-Language Support** - Search in one language, find results in any language
- **Speaker and Metadata Search** - Combined with speaker filtering and date ranges
- **Real-Time Indexing** - New transcriptions immediately searchable

**Model Tiers:**

Select the neural search model that best fits your needs:

- **Base** (Recommended) - Fast, low-resource, good general-purpose accuracy
- **Standard** - Balanced speed and accuracy for most use cases
- **Multilingual** - Best support for diverse language content
- **Specialized** - Domain-specific models for technical/legal/medical transcripts

**Easy Setup:**

Neural search is configured in the Admin UI with no code required:

1. Navigate to **Settings → Search**
2. Enable "Neural Search"
3. Select your preferred model tier
4. Click "Apply"

The system handles model downloads and indexing automatically.

**Performance Notes:**

- Initial indexing of existing transcripts may take time (background process)
- Queries are slightly slower than BM25-only (typically < 1 second for semantic queries)
- Hybrid search balances latency and relevance automatically

### Performance Improvements: 6x Faster Transcription

OpenTranscribe v0.4.0 changes the default Whisper model from `large-v2` to **`large-v3-turbo`**, delivering:

- **6x faster transcription** - Significantly reduced processing time
- **Reduced VRAM requirements** - ~6GB instead of ~10GB
- **Excellent English accuracy** - Only minor differences from large-v2
- **Automatic detection** - Large-v3-turbo auto-detects language

**Language-Specific Recommendations:**

| Language Use Case | Recommended Model | Why |
|---|---|---|
| English podcasts, meetings | `large-v3-turbo` | 6x faster, excellent accuracy |
| Spanish, French, German, Japanese | `large-v3-turbo` | Good accuracy, much faster |
| Thai, Cantonese, Vietnamese | `large-v3` | Turbo has slightly reduced accuracy |
| Translation to English | `large-v3` | **Turbo cannot translate** |
| Maximum accuracy (any language) | `large-v3` | Best overall accuracy |

**Model Configuration:**

Users can select their preferred model in **Settings → Transcription**:

```bash
# Or set in .env for all users
WHISPER_MODEL=large-v3-turbo  # Default - 6x faster
WHISPER_MODEL=large-v3        # For translation or maximum accuracy
WHISPER_MODEL=large-v2        # Legacy fallback
```

**Caching Improvements:**

v0.4.0 includes a new model caching system that automatically persists AI models between container restarts:

- Models automatically downloaded on first use
- Persistent across container rebuilds
- Simple `.env` configuration (`MODEL_CACHE_DIR`)
- Saves 2.6GB of bandwidth after first use
- Typical cold-start: 40-60 seconds, warm-start: < 5 seconds

## Upgrade Instructions

### Standard Update (Recommended)

For most users, upgrading is straightforward:

```bash
# Pull latest images
docker compose pull

# Restart services (migrations run automatically)
docker compose up -d

# Verify services are healthy
docker compose ps
```

### Development Environment

```bash
# Update and reset development environment
./opentr.sh restart-backend
./opentr.sh restart-frontend
```

### Optional: PyAnnote v4 Migration

The migration happens automatically on first use. No manual steps required.

To re-process existing videos with improved v4 diarization:

1. Navigate to a transcript
2. Click "Settings" → "Reprocess"
3. Select "Speaker Diarization" only
4. Submit

Processing will use PyAnnote v4 and complete 273x faster.

### Model Selection Guidance

**Choose `large-v3-turbo` if:**
- Primary language is English or major European languages
- Speed is important
- You have limited VRAM
- Most users won't use translation

**Choose `large-v3` if:**
- You need translation to English
- You process Thai, Cantonese, or other low-resource languages
- Maximum accuracy is critical
- You have sufficient VRAM (10GB+)

**Configuration:**

1. Users can select in **Settings → Transcription → Whisper Model**
2. Or set globally in `.env`: `WHISPER_MODEL=large-v3-turbo`
3. Default: `large-v3-turbo`

### Testing Recommendations

After upgrading, test these features:

- **Authentication** - Test your primary auth method (LDAP, OIDC, PKI, or local)
- **MFA** - Set up MFA and verify with authenticator app
- **Search** - Try semantic search queries
- **Transcription** - Upload a test video and verify it transcribes correctly
- **Speaker Diarization** - Verify speakers are detected and named correctly
